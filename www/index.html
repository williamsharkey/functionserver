The Vision: Your Computer in the Cloud

Function Server reimagines what a web application can be. Instead of isolated tools, you get a complete operating system with persistent storage, real file management, and a growing ecosystem of apps.

Every user gets their own sandboxed environment with local storage that persists like a real disk. Install apps, save files, customize your desktop. It's your machine, running in the browser.

  • ✓ Persistent localStorage disk image per user
  • ✓ Real terminal with SSH and Claude Code support
  • ✓ Visual app development with .layout and .ui files
  • ✓ Public sharing at /username/ URLs
  • ✓ AI assistants accessible from anywhere in the OS

App Architecture

A component-based system for building real applications

📄 Project Structure

Apps live in folders with multiple files. Layouts define screens, UI components are reusable, and the main .js file ties it together.

my-app/ |- main.js # App entry point |- screens/ | |- home.layout # Screen layouts | |- settings.layout |- components/ | |- button.ui # Reusable UI | |- card.ui |- app.json # Metadata

🔧 Designer Tool

Visual constraints-based layout editor. Drag components, set constraints (pin to edges, center, stack), preview on multiple sizes.

// Constraints example { "type": "button", "text": "Submit", "constraints": { "bottom": 20, "centerX": true, "width": "80%" } }

🎭 Craft Tool

Build reusable UI components. Define props, styles, and behavior. Components compile to efficient JavaScript.

// Component definition ALGO.component('MyButton', { props: ['text', 'onClick'], render: (props) => ` <button onclick="${props.onClick}"> ${props.text} </button>` });

📦 System Apps

Pre-installed apps managed by admins. Auto-update when the server updates. Users can fork to customize, or uninstall to hide.

// System apps loaded from /core/apps/ systemApps = [ { id: 'shade-station', ... }, { id: 'ticket-manager', ... }, { id: 'photobooth', ... } ];
