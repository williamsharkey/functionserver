Apps are artifacts in a shared VM

In most systems, apps are compiled binaries or isolated processes. You automate them from outside via WebDriver protocols. The AI is a client; the app is a server.

In FunctionServer, apps are the JavaScript. They're functions and objects living in the same VM that an AI agent can access via MCP.

When Claude calls getBoundingClientRect(), it's touching the same DOM element the user sees. When it patches window.openSubmenu, that's the real running function. The AI doesn't automate the OS—it inhabits it.

// Claude debugging a menu via MCP const menu = document.getElementById('programs-menu'); const rect = menu.getBoundingClientRect(); // Result: bottom is 40px too high // Fix it live, no refresh: menu.style.setProperty('bottom', '40px', 'important'); // User watches it snap into place

What this enables

Some things that emerge from apps and AI sharing the same address space:

AI with eyes

Claude can query element positions, check computed styles, inject test fixes, and verify they worked—before touching source files.

Morphing artifacts

The OS becomes a collection of artifacts that can be inspected, modified, and extended by anything with VM access.

Collapsed debug loop

Inspect live → measure → test fix → verify → commit. The browser becomes a REPL you can poke from anywhere.

Version the protocol

You can't version a morphing system. So each app becomes its own repo. What you version is the API and conventions.

The protocol defines the system

Apps declare inputs, outputs, and file types. The OS enforces conventions. Everything else is fluid.

ALGO API

Create windows, handle files, communicate via pubsub. The bridge between apps and OS.

MCP Endpoint

Any LLM with HTTP access can execute JavaScript, query DOM, click buttons, open apps.

File Type Registry

Apps declare what file types they handle. The OS routes, apps interpret.

Pubsub Channels

Apps communicate via named channels. Decoupled, async, observable.

The door is open

This is early. We don't know what this enables yet.
